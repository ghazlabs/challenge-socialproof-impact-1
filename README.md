# Challenge Social Proof & Impact - 1

Kamu sudah punya fundamental web development yang kuat. Kamu juga sudah belajar berbagai trick dan arts of the trade dari orang-orang yang berpengalaman di industri.

Sekarang saatnya kamu naik kelas.

Masalahnya, skill teknis saja belum cukup. Kode yang kamu tulis belum punya nilai apa-apa kalau belum benar-benar ngasih manfaat di dunia nyata. Justru dari situlah value kamu sebagai developer terbentuk. Semakin besar dampak yang dihasilkan dari kode yang kamu buat, semakin tinggi juga value kamu.

Karena itu, cara terbaik untuk berkembang adalah dengan menulis kode yang menyelesaikan masalah nyata. Masalah yang memang sudah ada dan dirasakan langsung oleh pengguna.

Challenge ini dibuat untuk melatih hal tersebut. Kamu akan belajar menemukan masalah di dunia nyata, lalu mengoding solusinya, dengan fokus pada impact, bukan sekadar kompleksitas.

## Tugas Kamu

Di challenge pertama ini, kita mulai dari langkah yang paling mudah dan relevan, yaitu kontribusi ke open source project.

Yang perlu kamu lakukan:

1. Cari beberapa open source project di GitHub dengan jumlah stars lebih dari 500 dan sesuai dengan keahlian coding kamu saat ini.
2. Buka bagian issue, lalu buat PR untuk issue yang ada. Kamu juga boleh bikin issue sendiri kalau kamu nemu bug di project tersebut.
3. Ulangi langkah di atas sampai kamu berhasil membuat minimal 3 PR.

   * Kamu boleh bikin beberapa PR di satu project yang sama.
   * Kamu juga boleh kontribusi ke beberapa project sekaligus.
   * Tapi kamu tidak boleh membuat semua PR hanya di satu project saja.
   * Artinya, minimal kamu harus berkontribusi ke 2 open source project berbeda.
4. Setelah semua selesai, isi [formulir ini](https://forms.gle/c6zLyA1JjuidoFQz7).

## Kriteria Penilaian

Yang dinilai di challenge ini bukan seberapa kompleks kode yang kamu tulis, tapi seberapa besar dampak dari kontribusi kamu.

Kriteria penilaiannya adalah:

1. Jumlah PR yang berhasil kamu buat (minimal 3).
2. Impact dari setiap PR yang kamu buat.

Impact tidak selalu berarti solusi yang rumit. Di open source project, sering banget ada issue yang solusinya cuma satu atau dua baris kode, tapi dampaknya besar buat end user.

Contohnya, kamu kontribusi ke web app berupa dashboard monitoring. Ternyata ada issue saat diakses dari mobile, tampilannya berantakan karena belum ada tailwind classes yang sesuai. Solusinya sederhana, cukup menambahkan class yang kurang. Tapi dampaknya besar karena sekarang dashboard tersebut bisa diakses dengan nyaman lewat mobile.

Kasus seperti ini sering terjadi karena menjaga open source project itu tidak mudah. Issue kecil itu wajar, dan justru sering kali issue seperti inilah yang punya impact besar buat pengguna.
